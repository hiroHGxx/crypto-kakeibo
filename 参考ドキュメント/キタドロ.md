## 1. 事前準備

共通フォーマットを作るために、以下6つの作業を行います 。

### ① Gtaxから共通フォーマットのサンプルをダウンロード

* Gtaxにログインし、「データ取込」→「対応外データ」→「【一般用】共通フォーマットをダウンロード」をクリックします 。


* パソコンに「共通フォーマットのサンプル」がダウンロードされればOKです 。



### ② 共通フォーマットをGoogleスプレッドシートにアップロード

* Googleスプレッドシートを開き、「ファイル」→「インポート」→「アップロード」を選択します 。


* 共通フォーマットのサンプル（CSV形式）をアップロードし、「データをインポート」をクリックします 。



### ③ Etherscanからウォレットの取引明細をGET

* Etherscanの検索窓にウォレットアドレスを入力し、取引履歴を表示させます 。


* 右下の「Download: CSV Export」をクリックし、開始日・終了日を指定してダウンロードします 。


* 家計簿作成には以下4つのデータが必要です 。


* 
**Transactions**: トランザクションのデータ 


* 
**Internal Transactions**: 内部トランザクション・スマートコントラクト間で発生する取引データ 


* 
**Token Transfers (ERC-20)**: ERC-20トークンのデータ 


* 
**NFT Transfers (ERC-721 & ERC-1155)**: NFTのデータ 





### ④ 取引明細をGoogleスプレッドシートにアップロード

* スプレッドシートに新しいシートを追加し、EtherscanからダウンロードしたCSVをインポート（「現在のシートに追加する」を選択）します 。


* Gtax提出用の「本番用」と「下書き用」のタブを分けると作業しやすいです 。



### ⑤ 不要な項目を削除

* Etherscanのデータから、以下の不要な列を削除します 。


* Transaction Hash, Blockno, Unix Timestamp, From, To, ContractAddress, CurrentValue @ $.../Eth, TxnFee(USD), Historical $Price/Eth 





### ⑥ 共通フォーマットで使えるようにカスタマイズ

1行目の英語項目を、Gtax共通フォーマットの日本語項目に書き換えます 。

* 
`DateTime (UTC)` → そのまま（後でJSTに変更） 


* 
`Value_IN(ETH)` → 取引量(+) 


* 
`Value_OUT(ETH)` → 取引量(-) 


* 
`TxnFee(ETH)` → 手数料数量 


* 
`Status` → 状態 


* 
`ErrCode` → エラーコード 


* 
`Method` → 作業内容 



#### 各列の記入ポイント

* 
**A列 取引所名**: 取引した場所（OpenSea, MetaMask, Mint siteなど） 。


* 
**B列 日時(JST)**: 日本時間（UTCに9時間プラス） 。


* 
**C列 取引種別**: 取引の内容に応じて以下を指定 。


* 
`売買`: NFT・仮想通貨同士の交換（スワップ） 


* 
`送付`: ウォレットの送金・送信 


* 
`支払`: 仮想通貨・NFTで商品を購入・サービス利用料を支払う 


* 
`経費`: 広告費を仮想通貨で支払い経費計上 


* 
`減少`: Giveaway開催でNFTを送信、仮想通貨/NFTのGOXなど 


* 
`手数料`: 仮想通貨/NFTは送金せず、手数料のみの支払い 


* 
`ボーナス`: 市場価値のある仮想通貨やNFTをエアドロップでもらう（利益計上） 


* 
`無償取得`: 市場価値のないNFTをフリーミント・Giveawayでもらう 




* 
**D列 取引通貨名(+)**: 購入・獲得した通貨名やNFTコレクション名（非対応通貨は「NFT資産」と記入） 。


* 
**E列 取引量(+)**: 獲得した数量 。


* 
**F列 取引通貨名(-)**: 支払った通貨・売却したNFT名 。


* 
**G列 取引量(-)**: 支払った数量 。


* 
**H列 取引額時価**: 非対応アルトコインやNFTの時価評価用（原則日本円で記入） 。


* 
**I列 手数料通貨名**: ガス代の通貨名 。


* 
**J列 手数料数量**: 支払ったガス代の数量 。

## 2. Gtax 共通フォーマットの入力方法 (パターン別)

※本マニュアルは著者個人の調べに基づくものであり、税務の最終判断は税務署や税理士にご相談ください 。

### ① NFTを1作品買って仮想通貨を支払った

* 
**パターンA（購入時のガス代を取得価格にプラスする）** ※推奨例 


* 取引種別: 売買 


* 取引通貨名(+): NFT資産CNP001 / 取引量(+): 1 


* 取引通貨名(-): ETH / 取引量(-): NFTの価格 + 購入時のガス代 




* 
**パターンB（ガス代を購入年度の経費にする）** 


* 取引種別: 売買 


* 取引通貨名(-): ETH / 取引量(-): NFTの価格 


* 手数料通貨名: ETH / 手数料数量: 支払ったガス代 





### ② NFTを1度に2作品買った

* 複数作品をまとめて購入した場合、履歴を2行に分けて記入し、ガス代は「2」で割ってそれぞれにプラスします 。


* 取引量(-): NFTの価格 + (ガス代 ÷ 2) 





### ③ NFTをクレジットカードで買った

* 取引通貨名(-): JPY / 取引量(-): 日本円で支払額を記入 。



### ④ NFTを売って仮想通貨を受け取った

* 取引種別: 売買 


* 取引通貨名(+): ETH / 取引量(+): ウォレットに入金された金額 


* 取引通貨名(-): NFT資産CNPJ001 / 取引量(-): 1 


* 初回出品時などに支払ったガス代（Set Approval For All等）は別行で「取引種別: 手数料」として経費計上します 。



### ⑤ NFTを友人に日本円で売った

* 取引通貨名(+): JPY / 取引量(+): 受け取った日本円金額 。


* 手数料が発生した場合は、手数料通貨名に「JPY」、数量に金額を記入 。



### ⑥ フリーミントでNFTをもらいガス代を払った

* 取引種別: 売買 


* 取引通貨名(+): NFT資産CNP001 / 取引量(+): 1 


* 取引通貨名(-): ETH / 取引量(-): 支払ったガス代（購入価格0円＋ガス代という扱い） 



### ⑦ 市場価値がないNFTを無料でもらった

* 販売前のNFTやSBT、非売品などをもらった場合 。


* 取引種別: 無償取得 


* 取引通貨名(+): NFT資産KTDR001 / 取引量(+): 1 



### ⑧ NFTをガチホ用ウォレットに送信した

* 自分のウォレット間の移動は損益が発生しないため、NFTの送信履歴自体は記入不要です 。


* 送信時に支払ったガス代のみ記入します 。


* 
**経費に含める場合**: 取引種別「手数料」、手数料通貨名「ETH」、数量にガス代 。


* 
**経費に含めない場合**: 取引種別「減少」、取引通貨名(-)「ETH」、数量(-)にガス代 。





### ⑨ NFTをGiveaway（プレゼント）した

* 
**パターンA（経費にする／0円で売却）**: 取引種別「売買」、(+)通貨「JPY」、(+)量「0」、(-)通貨「NFT資産」、(-)量「1」、手数料にガス代 。


* 
**パターンB（経費にしない／ポジションを減らす）**: 2行に分ける。1行目は取引種別「減少」でNFTを1減らす。2行目は取引種別「減少」でETH（ガス代）を減らす 。


* 
**パターンC（経費計上）**: プロモーション目的などの場合。取引種別「経費」、(-)通貨「NFT資産」、(-)量「1」、手数料にガス代 。



### ⑩ NFTをGiveawayでもらった

* 価値のあるNFTをもらった場合、課税の対象（雑所得、贈与税、一時所得のいずれか）になります 。


* いずれの場合も共通フォーマットの書き方は同じです。
* 取引種別: ボーナス 


* 取引通貨名(+): NFT資産CNP001 / 取引量(+): 1 


* 取引額時価: 日本円の時価を記入（例: 150000） 




* **時価の調べ方**:
* A: 送信者の「NFTの購入価格」 


* B: もらった時点の「NFTのフロア価格」 


* C: 直近の「NFTの平均売買価格」を計算し、ETH価格から日本円に換算 





### ⑪ 自分のウォレット間で仮想通貨を移動させた

* 送金履歴の記入は不要 。


* ガス代のみ記入。取引種別「減少」、取引通貨名(-)「ETH」、取引量(-)にガス代 。



### ⑫ 仮想通貨取引所へ仮想通貨を送金した

* ⑪と同様、送金履歴は不要でガス代のみ記入 。


* 経費にする場合は取引種別「経費」とします 。



### ⑬ 仮想通貨で商品を購入した

* 仮想通貨を売却して利益確定したとみなされます 。


* 取引種別: 支払 


* 取引通貨名(-): BTC / 取引量(-): 支払った額 / 手数料にガス代 



### ⑭ ウォレットに仮想通貨がエアドロップされた

* 取引種別: ボーナス 


* 取引通貨名(+): PLSなど / 取引量(+): 数量 


* 新しい通貨等でGtaxが時価を取得できない場合は「取引額時価」に日本円を手動入力します 。



### ⑮ 詐欺被害にあった

* 被害に遭った場合は警察に届け出ます 。「雑損控除」の対象になる可能性がありますが、詐欺の場合は対象外となることもあります 。


* 
**損失計上する場合**: 取引種別「損失」、取引通貨名(-)「NFT資産」、数量(-)「1」 。


* 
**損失計上できない場合**: 取引種別「減少」で処理 。



### ⑯ よくわからないNFTがウォレットに送られていた

* 購入していないスキャムNFT等は、記入不要（無視）か、取引種別「ボーナス」で時価「0」として処理します 。



### ⑰ 経費として仮想通貨を支払った

* 取引種別: 経費 


* 取引通貨名(-): BTC / 取引量(-): 数量 



### ⑰ 手数料として仮想通貨を支払った

* 取引種別: 手数料 / 手数料通貨名: BTC / 手数料数量: 数量 。


* 経費にしない場合は取引種別「減少」で処理します 。



### ⑲ 仮想通貨同士を交換(スワップ)した

* 取引種別: 売買 


* 取引通貨名(+): 取得した通貨（SOL） / 量(+): 取得量 


* 取引通貨名(-): 支払った通貨（ME） / 量(-): 支払量 



### ⑳ MetaMaskで仮想通貨（クレジットカード等）を買った

* 取引種別: 売買 


* 取引通貨名(+): ETH / 量(+): 1 


* 取引通貨名(-): JPY / 量(-): 日本円の支払額 



### ㉑ クリック戦争に敗北した

* ガス代のみ発生。経費にしない場合は「減少」、経費にする場合は「経費」として、(-)側にETHとガス代を記入します 。



### ㉒ コントラクト移行（CNPなど）

* 古いNFT（価値あり）は将来売却する時のためにそのまま残し、新しくエアドロップされたNFT（実質0円）を無償取得として記録します 。


* 新NFT: 取引種別「無償取得」 / 取引通貨名(+)「NFT資産CNP売れない」 / 取引量(+)「1」 。



### ㉓ ブリッジした（Ethereum → Astar zk EVM など）

* 異なるチェーンへの移動は、「送付」と「預入」を組み合わせて記入します 。


* 1行目: 取引種別「送付」 / 取引通貨名(-)「ETH」 。


* 2行目: 取引種別「預入」 / 取引通貨名(+)「ETH」 。


* ガス代が複数回発生した場合は、それぞれ「減少」（または「経費」）で処理し、ETH残高にズレが出ないようにします 。



### ㉔ チェーン移行に伴う送金（例: BONSAICOIN）

* 送金時の「支払ったガス代」と「プレゼントされたボーナス分（1%分など）」の2つの取引を記入します 。


* ガス代: 取引種別「減少」（または「経費」） 。


* ボーナス: 取引種別「ボーナス」、取引通貨名(+)「BONSAICOIN」、取引額時価に日本円を入力 。

## 3. 家計簿が完成したあとにやることリスト

1. **共通フォーマットをパソコンにダウンロード**
スプレッドシートの「ファイル」→「ダウンロード」から `.xlsx` または `.csv` 形式でダウンロードします 。


2. **Gtaxに共通フォーマットをアップロード**
Gtaxの「データ取込」→「対応外データ」→「【一般用】共通フォーマットを取り込む」からアップロードします 。


3. **計算の確定**
「取引一覧」画面で入力内容にミスがないか確認し、「未確定」→「2023年度の計算結果を確定する」→「確定」と進めます 。


4. **取引結果をダウンロード**
「ポートフォリオ」から「詳細な情報をダウンロード」をクリックし、結果サマリーをダウンロードします 。


5. **確定申告書に入力**
結果サマリーに記載された「実現損益（雑所得など）」や「経費」の金額を、国税庁の確定申告書作成コーナーに入力します 。


6. **取引明細は必ず保存**
確定申告で使用した資料や計算根拠となるファイルは、パソコン等にフォルダを作って7年間保存してください 。
